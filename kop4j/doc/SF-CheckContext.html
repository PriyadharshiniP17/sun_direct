Execution Context Checks

General description

KOP4J gives the opportunity to inject different checks on the Android java environment.
Those checks are positioned in the code at the choice of the user.

The check of the execution context consists to check that the App is not rooted, tampered by Frida or Xposed framework, executed on an emulator or being debugged.

Warning

These checks must be inserted in classes that are direct or indirect subclasses of the class Context (e.g. Activity, ActivityGroup, Application, ContextWrapper...)

Security note

Security

Environments checks could impact a bit the performance. It is important to apply those checks at the start-up of the app , but not only.
An hacker could try to by pass the check!
Then it could be smart to execute some checks during the runtime of the app or before some critical operation.

Protection Configuration

Without reaction selection

To inject a check in a method, the operator will set in the .pro file the check command followed by the method designator

Execution context : -kop-check-exe-context <classic method designator >

Example : -kop-check-exe-context public class package.name.Main {public static void main(...);}

With reaction selection

KOP provides 4 reactions:

  • Immediate: if an anomaly is detected, the app fails immediately.
    -kop-check-exe-context,immediate <classic method designator >

  • Delayed-Strong: if an anomaly is detected, the app fails with a certain delay. This mechanism is described in Delayed Reaction chapter
    -kop-check-exe-context,delayed-strong <classic method designator >

  • Delayed: Same protection as the previous one, but less restrictive. This mechanism is described in Delayed Reaction chapter
    -kop-check-exe-context,delayed <classic method designator >

  • NoReaction: if an anomaly is detected, the app continues to run.
    -kop-check-exe-context,no-reaction <classic method designator >

If the reaction is not specified, KOP applies the immediate reaction.

Example : -kop-check-exe-context,immediate, public class package.name.Main {public static void main(...);}

will fail instantly if an anomaly is detected.
Note that it is equivalent to the first exemple as the default reaction is the immediate reaction.

Error messages

K7 : "Bitmap value" : Environment context is not safe.

Bitmap value : Decimal value. To convert into bitmap. Attack detected.

bit Mask Binary Mask Hexa Signification
4 00010000b 0x10 Root is detected.
5 00100000b 0x20 Frida is detected.
6 01000000b 0x40 XPosed is detected.
7 10000000b 0x80 an Emulator is detected.
8 100000000b 0x100 a debugger is detected
