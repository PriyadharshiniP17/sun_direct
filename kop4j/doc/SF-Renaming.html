Renaming

General description

A part of the obfuscation consist to rename the methods and fields. This techniques has two advantages:

  • It makes the reverse more complicated
  • It reduce the size of the application

KOP4J has implemented different strategy of renaming are available:

  • Alphabet case sensitive classical strategy present on ProGuard and R8: AabBcC
  • Alphabet case insensitive : aabbcc
  • UTF8 is the strategy selected by default based and designed to generate confusion :OO0o0

The renaming can be disabled mainly to keep the link with external dependency

Security note

Security

The UTF8 is probably the strategy the most difficult to analyze

Protection Configuration

Renaming startegy

The command to fix a new strategy is -kop-rename-strategy strategyName The strategy name can be alphabet-case-sensitive, alphabet-case-insensitive or utf8

for instance -kop-rename-strategy alphabet-case-sensitive

will apply the renaming with case sensitive letters

keep naming

The command -keep <class designator> will prevent code stripping and renaming for classes and class members
The command -keepnames <class designator> will prevent renaming for classes and class members

The command -keepclassmembers <class designator> will prevent code stripping and renaming for class members only
The command -keepclassmembernames <class designator> will prevent renaming for class members only

The command -keepclasseswithmembers <class designator> will prevent code stripping and renaming for classes and class members, if class members present
The command -keepclasseswithmembernames <class designator> will prevent renaming for classes and class members, if class members present
