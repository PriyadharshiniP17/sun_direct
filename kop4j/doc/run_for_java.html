Running KOP4J on plain Java and Android projects

KOP4J can obfuscate plain Java projects that are already packaged as .jars, or as plain directories containing .class files.

Input files

Input files are specified using the -injars <path> option in proguard-rules.pro.
Input files are the files to be protected.

The path can be:

  • A path to a java .jar file.
  • A path to a directory containing .class files.

Example: protect the contents of myapp.jar.

-injars myapp.jar

Input files, filtered

Optional filters can be applied to the contents specified by the -injars option proguard-rules.pro.
These optional filters should immediately follow -injar's <path>, and be specified between parenthesis.

Example: protect the contents of myapp.jar, but only consider the contents of the mypackage subdirectory.

-injars myapp.jar(mypackage/**.class)

Output files

Output files are specified using the -outjars <path> option.

The path can be:

  • A path to a java .jar file.
  • A path to a directory that will contain .class files.

Warning

The output type is based on extension: it is not possible to create an output directory with the .jar suffix.

Output files, filtered

Optional filters can be applied to the contents specified in the -outjars option in proguard-rules.pro.
The syntax is identical to that of the -injars option.

Library files

Library files are specified using the -libraryjars <path> option.
Library files are files that are dependencies of the input files.
Library files are not protected.

The path can be;

  • A path to a java .jar file.
  • A path to a directory containing .class files.

Example: indicate the file library.jar as a non-protected dependency.

-libraryjars library.jar

Library files, filtered

Optional filters can be applied to the contents specified by the -libraryjars option.
These optional filters should immediately follow -libraryjar's <path>, and be specified between parenthesis.

Example: use the contents of library.jar, but only consider the contents of the mypackage subdirectory as dependencies.

-libraryjars library.jar(mypackage/**.class)

Input/Output, examples

Multiple -injars and -outjars can be combined to rearrange the contents of an application.

Merging contents

This can be done using multiple -injars and a single -outjar.

Example: Read the contents of myapp.jar and mylib.jar , consolidate to myoutput.jar

-injars myapp.jar -injars mylib.jar -outjar myoutput.jar

Hash strategy option

By default, the hash strategy is using external AnNaNaS library, only delivered for Android targets.
To use the pure java implementation (no external dependency), you will need to add the following option in kop-rules.pro when invoking KOP4J : 

-kop-hash-strategy java
