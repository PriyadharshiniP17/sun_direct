Compatibility with ProGuard

KOP4J aims to be compatible with the ProGuard open source tool's command line parameters.

It DOES NOT reuse any of ProGuard's source code.

Compatibility with Firebase Performance SDK

When using the Firebase Performance SDK, there are 2 requirements:

  1. Include these 2 rules in the KOP4J rule file
    -kop-exclude com.google.firebase.perf.network.NetworkRequestMetricBuilderUtil
    -kop-exclude com.google.protobuf.MessageSchema
  2. Do not enable reflexivity on Firebase package
    If reflexivity was enabled globally, the following rule can be added as the last one (it will override the previous rules) :
    -kop-accessthroughreflection,disable class com.google.firebase.perf.** { *; }
