Installer Checks

General description

KOP4J gives the opportunity to check what installer was used to install the application.
Those checks are positioned in the code at the choice of the user.

The installer check : consists in checking that the name of the application installer package matches a set of expected values.

Warning

These checks must be inserted in classes that are direct or indirect subclasses of the class Context (e.g. Activity, ActivityGroup, Application, ContextWrapper...)

Protection Configuration

Without reaction selection

To inject a check in a method, the operator will set in the .pro file the check command followed by the method designator

Installer : -kop-check-installers,installers:<package1|package2> <classic method designator >

Example : -kop-check-installers,installers:com.android.vending public class package.name.Main {public static void main(...);}

will inject a check that the app can only be installed from the Google Play Store, but not from any other store (such as the Amazon AppStore).

With reaction selection

KOP provides 4 reactions:

  • Immediate: if an anomaly is detected, the app fails immediately.
    -kop-check-installers,immediate <classic method designator >

  • Delayed-Strong: if an anomaly is detected, the app fails with a certain delay. This mechanism is described in Delayed Reaction chapter
    -kop-check-installers,delayed-strong <classic method designator >

  • Delayed: Same protection as the previous one, but less restrictive. This mechanism is described in Delayed Reaction chapter
    -kop-check-installers,delayed <classic method designator >

  • NoReaction: if an anomaly is detected, the app continues to run.
    -kop-check-installers,no-reaction <classic method designator >

If the reaction is not specified, KOP applies the immediate reaction.

Example : -kop-check-installers,installer:com.android.vending public class package.name.Main {public static void main(...);}
will inject a check that the installer is the official Google Play Store and fails instantly if an anomaly is detected.
Note that it is equivalent to the first exemple as the default reaction is the immediate reaction.

With installer selection

If no installers are indicated, KOP defaults to com.android.vending|com.amazon.*, which matches the Google Play Store and the Amazon AppStore.

Warning

The values for the default installers, and those supplied in this documentation are dependent on the store providers.
While they should be correct and stable, you must validate that they match your chosen store before deployment.

Multiple allowed installers can be checked, by separating their package with a |.
A * at the end of a package is used as a wildcard for this package.

Typical installers could be, for example: - com.android.vending: The Google Play Store - com.amazon.*: Amazon AppStore - com.sec.android.app.samsungapps: Samsung Galaxy Store - com.huawei.appmarket: Huawei AppGallery - com.xiaomi.market: Xiaomi App Store - null: Special placeholder, that allows non-existent package installer information. only use for debugging.

Security

Choose the allowed installers based on where you plan to deploy your app.

The installers:null value effectively allows installation of an apk with the adb tool, and should only be used for debugging.

Error messages

K12 : The package installer does not match the expected one.

Example crash message

K12 : com.apkmirror.app

The app was installed from the apkmirror app, and not from the official playstore.
